Introduction Last updated: 2019-06-01

La aplicación de Andalucía Descubre está no solo orientada a los usuarios finales, sino también a programadores que deseen integrar la información de la misma en sus propias aplicaciones. Para ello, ponemos a disposición de los desarrolladores una API REST que permite acceder a la información de Andalucía DEscubre de forma estructurada y en tiempo real, además de una documentación que explica el frontal de la aplicación y cómo se ha desarrollado para servir de guía a los programadores que deseen integrar la información de Andalucía Descubre en sus propias aplicaciones.

Tecnologías usadas en el proyecto

El proyecto de Andalucía Descubre ha sido desarrollado con las siguientes tecnologías:

Frontend

  • Angular: Framework de desarrollo de aplicaciones web que utiliza Typescript como lenguaje de programación.
  • Typescript: Lenguaje de programación que añade capacidades de programación orientada a objetos a Javascript.
  • SCSS (Sass): Preprocesador CSS que añade funcionalidades como variables, mixins y anidamiento.
  • Bootstrap 5: Framework de diseño web que facilita la creación de interfaces web.
  • Font Awesome: Biblioteca de iconos que facilita la inclusión de iconos en las aplicaciones web.
  • Node: Entorno de ejecución de Javascript que permite ejecutar Javascript en el servidor.
  • NPM: Gestor de paquetes de Node que facilita la instalación de paquetes y dependencias.

Backend

  • FastAPI: Framework de desarrollo de APIs web con Python.
  • Python: Lenguaje de programación utilizado para desarrollar la API.
  • MySQL: Sistema de gestión de bases de datos relacionales.
  • PIP: Gestor de paquetes de Python que facilita la instalación de paquetes y dependencias.
  • Uvicorn: Servidor ASGI que permite ejecutar aplicaciones web desarrolladas con FastAPI.

Diseño del proyecto

El proyecto de Andalucía Descubre tiene como finalidad principal ofrecer a los usuarios información sobre los lugares más emblemáticos de Andalucía, así como facilitar la planificación de rutas turísticas y la reserva de alojamientos y actividades en la región. El diseño del proyecto se ha realizado siguiendo las mejores prácticas de diseño web y desarrollo de aplicaciones, con el objetivo de ofrecer una experiencia de usuario intuitiva y atractiva. Cuenta con un buscador de lugares, un mapa interactivo con los puntos de interés de cada localidad y un filtrado por provincias para facilitar la búsqueda de información.

Despliegue en local

La aplicación de Andalucía Descubre está disponible en la web y se puede acceder a ella desde cualquier dispositivo con conexión a Internet. Pero si deseas desplegar la aplicación en tu propio servidor, a continuación te mostramos cómo hacerlo.

Despliegue de la API

Para desplegar la API de Andalucía Descubre en tu propio servidor, sigue los siguientes pasos:

  1. Clona el repositorio de la API de Andalucía Descubre desde GitHub.
  2. Instala las dependencias de la API utilizando el gestor de paquetes de Python PIP.
  3. Configura la conexión a la base de datos MySQL en el archivo de configuración de la API.
  4. Crea las tablas necesarias en la base de datos MySQL.
  5. Arranca el servidor de la API utilizando el comando de Uvicorn.
Comandos:
								
git clone (url del repositorio)

pip install -r requirements.txt

uvicorn main:app --reload

Despliegue del Front

Para desplegar el frontend de Andalucía Descubre en tu propio servidor, sigue los siguientes pasos:

  1. Clona el repositorio del frontend de Andalucía Descubre desde GitHub.
  2. Instala las dependencias del frontend utilizando el gestor de paquetes de Node NPM.
  3. Configura la URL de la API en el archivo de configuración del frontend.
  4. Arranca el servidor de desarrollo del frontend utilizando el comando de Angular.
Comandos:
								
git clone (url del repositorio)

npm install

ng serve

APIs

La API de Andalucía Descubre está diseñada para ser fácil de usar y flexible, permitiendo a los desarrolladores acceder a la información de la aplicación de forma estructurada y en tiempo real. La API cuenta con varios endpoints que permiten acceder a la información de los lugares más emblemáticos de Andalucía, así como a la información de los eventos, platos y monumentos de la región. En esta sección encontrarás una descripción detallada de los endpoints de la API, así como ejemplos de las respuestas que devuelven.

/api (TOWNS)

El endpoint /api de la API de Andalucía Descubre permite acceder a la información de los lugares más emblemáticos de Andalucía, así como a la información de los eventos, platos y monumentos de la región. El endpoint /api cuenta con los siguientes subendpoints:

  • /towns (POST): Permite crear una nueva localidad.
  • /towns (GET): Permite obtener la información de una todas las localidades.
  • /towns/{id} (GET): Permite obtener la información de una localidad en concreto.
  • /towns/{id} (PUT): Permite actualizar la información de una localidad en concreto.
  • /towns/{id} (DELETE): Permite eliminar una localidad en concreto.
  • /towns/{id}/events (GET): Permite obtener la información de los eventos de una localidad en concreto.
  • /towns/{id}/events (POST): Permite crear un nuevo evento en una localidad en concreto.
  • /towns/{id}/events (DELETE): Permite eliminar un evento en una localidad en concreto.
  • /towns/{id}/dishes (GET): Permite obtener la información de los platos de una localidad en concreto.
  • /towns/{id}/dishes (POST): Permite crear un nuevo plato en una localidad en concreto.
  • /towns/{id}/dishes (DELETE): Permite eliminar un plato en una localidad en concreto.
  • /towns/{id}/monuments (GET): Permite obtener la información de los monumentos de una localidad en concreto.
  • /towns/{id}/monuments (POST): Permite crear un nuevo monumento en una localidad en concreto.
  • /towns/{id}/monuments (DELETE): Permite eliminar un monumento en una localidad en concreto.

/auth (USERS)

El endpoint /auth de la API de Andalucía Descubre permite gestionar la autenticación y la administración de usuarios. El endpoint /auth cuenta con los siguientes subendpoints:

  • /auth/register (POST): Permite registrar a un nuevo usuario.
  • /auth/login (POST): Permite autenticar a un usuario.
  • /auth/user-info (GET): Permite obtener la información del usuario autenticado.
  • /auth/users (GET): Permite obtener la información de todos los usuarios (requiere permisos de administrador).
  • /auth/admin/login (POST): Permite autenticar a un administrador.
  • /auth/user/{user_id} (PUT): Permite actualizar la información de un usuario en concreto (requiere permisos de administrador).
  • /auth/user/{user_id} (DELETE): Permite eliminar a un usuario en concreto (requiere permisos de administrador).
  • /auth/logs (GET): Permite obtener los registros de acciones (requiere permisos de administrador).
  • /auth/send-email (POST): Permite enviar un correo electrónico.

Securización

La API de Andalucía Descubre cuenta con un sistema de autenticación basado en tokens JWT (JSON Web Tokens) que permite proteger los endpoints de la API y garantizar que solo los usuarios autorizados puedan acceder a la información. El sistema de autenticación de la API de Andalucía Descubre se basa en los siguientes principios:

JWT

El sistema de autenticación de la API de Andalucía Descubre utiliza tokens JWT (JSON Web Tokens) para proteger los endpoints de la API y garantizar que solo los usuarios autorizados puedan acceder a la información. Los tokens JWT son cadenas de texto codificadas que contienen información sobre el usuario autenticado, así como una firma digital que garantiza la integridad del token. Para autenticar a un usuario, la API de Andalucía Descubre genera un token JWT que se envía al cliente, el cual debe incluirlo en las cabeceras de las peticiones a los endpoints protegidos. El token JWT tiene una duración limitada y debe ser renovado periódicamente para garantizar la seguridad de la aplicación.

Ejemplo de token JWT:
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJ1c2VyX2lkIjoxMjMsImVtYWlsIjoidXN1YXJpb0BleGFtcGxlLmNvbSIsInRvd25faWQiOjEsImRhdGVzIjp7

CORS

La API de Andalucía Descubre utiliza la política de CORS (Cross-Origin Resource Sharing) para permitir que los clientes web accedan a los recursos de la API desde un dominio diferente. La política de CORS de la API de Andalucía Descubre se basa en los siguientes principios:

  • La API solo permite peticiones POST, PUT y DELETE desde el dominio del frontend.
  • La API incluye las cabeceras CORS necesarias en las respuestas para permitir el acceso desde el dominio del frontend.
  • La API verifica el origen de las peticiones y rechaza las peticiones de dominios no autorizados.
Ejemplo de cabeceras CORS:
Access-Control-Allow-Origin: https://frontend.example.com
Access-Control-Allow-Methods: POST, PUT, DELETE
Access-Control-Allow-Headers: Content-Type, Authorization
Access-Control-Allow-Credentials: true

IP Whitelisting

La API de Andalucía Descubre utiliza una lista blanca de direcciones IP para restringir el acceso a los endpoints protegidos y garantizar que solo los clientes autorizados puedan acceder a la información. La lista blanca de direcciones IP de la API de Andalucía Descubre se basa en los siguientes principios:

  • La API solo permite peticiones desde las direcciones IP autorizadas.
  • La API verifica la dirección IP de las peticiones y rechaza las peticiones de direcciones no autorizadas.
  • La API incluye las cabeceras necesarias en las respuestas para permitir el acceso desde las direcciones IP autorizadas.

Despliegue

En esta sección se detalla como está desplegada la aplicación de Andalucía Descubre en la web, en este caso está en un servidor privado contratado por el equipo de desarrollo, la aplicación está desplegada en un servidor Apache con una base de datos MySQL y también hay una securización del servidor.

Servidor

La aplicación de Andalucía Descubre está desplegada en un servidor privado contratado por el equipo de desarrollo, el servidor cuenta con las siguientes características:

  • Servidor Apache (Para servir las páginas web de la aplicación).
  • Base de datos MySQL (Para almacenar la información de la aplicación).
  • Seguridad SSL. (Es un protocolo de seguridad que permite que los datos se transmitan de forma segura entre el servidor y el cliente).
  • Firewall (Es un sistema de seguridad que protege el servidor de posibles ataques).
  • Protección DDoS (Es un sistema de protección que evita los ataques de denegación de servicio).

Base de datos

La base de datos de Andalucía Descubre está alojada en un servidor MySQL, el gestor de la base de datos es accesible a través de un navegador web y permite gestionar la información de la aplicación de forma sencilla y rápida, pero para su seguridad, solo permite el acceso desde las IP que estén en la lista blanca. La base de datos de Andalucía Descubre cuenta con las siguientes tablas:

  • AD_USERS: Almacena la información de los usuarios de la aplicación.
  • AD_TOWNS: Almacena la información de las localidades de Andalucía.
  • AD_EVENTS: Almacena la información de los eventos de las localidades.
  • AD_DISHES: Almacena la información de los platos típicos de las localidades.
  • AD_MONUMENTS: Almacena la información de los monumentos de las localidades.
  • AD_LOGS: Almacena los registros de acciones de los usuarios.
Ejemplo de estructura de la tabla AD_USERS:
CREATE TABLE AD_USERS (
	userID INT PRIMARY KEY AUTO_INCREMENT,
	email VARCHAR(255) NOT NULL,
	password VARCHAR(255) NOT NULL,
	townID INT,
	FOREIGN KEY (townID) REFERENCES AD_TOWNS(townID)
);
Ejemplo de estructura de la tabla AD_TOWNS:
CREATE TABLE AD_TOWNS (
	townID INT PRIMARY KEY AUTO_INCREMENT,
	townName VARCHAR(255) NOT NULL,
	townDescription TEXT NOT NULL,
	townImage VARCHAR(255) NOT NULL,
	townMap VARCHAR(255) NOT NULL
);
Ejemplo de estructura de la tabla AD_EVENTS:
CREATE TABLE AD_EVENTS (
	eventID INT PRIMARY KEY AUTO_INCREMENT,
	townID INT,
	eventName VARCHAR(255) NOT NULL,
	eventDescription TEXT NOT NULL,
	eventDate DATE NOT NULL,
	FOREIGN KEY (townID) REFERENCES AD_TOWNS(townID)
);
Ejemplo de estructura de la tabla AD_DISHES:
CREATE TABLE AD_DISHES (
	dishID INT PRIMARY KEY AUTO_INCREMENT,
	townID INT,
	dishName VARCHAR(255) NOT NULL,
	dishDescription TEXT NOT NULL,
	dishImage VARCHAR(255) NOT NULL,
	FOREIGN KEY (townID) REFERENCES AD_TOWNS(townID)
);
Ejemplo de estructura de la tabla AD_MONUMENTS:
CREATE TABLE AD_MONUMENTS (
	monumentID INT PRIMARY KEY AUTO_INCREMENT,
	townID INT,
	monumentName VARCHAR(255) NOT NULL,
	monumentDescription TEXT NOT NULL,
	monumentImage VARCHAR(255) NOT NULL,
	FOREIGN KEY (townID) REFERENCES AD_TOWNS(townID)
);
Ejemplo de estructura de la tabla AD_LOGS:
CREATE TABLE AD_LOGS (
	logID INT PRIMARY KEY AUTO_INCREMENT,
	userID INT,
	userIP VARCHAR(255) NOT NULL,
	logDate DATETIME NOT NULL,
	description TEXT NOT NULL,
	type ENUM('register', 'login', 'logout', 'update', 'delete') NOT NULL,
	FOREIGN KEY (userID) REFERENCES AD_USERS(userID)
);

VPS

La aplicación de Andalucía Descubre está desplegada en un servidor VPS (Virtual Private Server) contratado por el equipo de desarrollo, el servidor VPS está securizado tanto para el acceso a la consola como para el ssh, utiliza un sistema de autenticación basado en claves públicas y privadas, una passphare, un sistema de autenticación de dos factores y un sistema de backups automáticos.

Características del servidor VPS:
  • Seguridad SSL. (Es un protocolo de seguridad que permite que los datos se transmitan de forma segura entre el servidor y el cliente).
  • Firewall (Es un sistema de seguridad que protege el servidor de posibles ataques).
  • Protección DDoS (Es un sistema de protección que evita los ataques de denegación de servicio).
  • Autenticación basada en claves públicas y privadas.
  • Passphare para la autenticación de claves privadas.
  • Sistema de autenticación de dos factores.
  • Sistema de backups automáticos.

Funcionalidades

La versión móvil de Andalucía Descubre cuenta con las siguientes funcionalidades:

  • Interfaz intuitiva y fácil de usar.
  • Acceso a la información de las localidades, eventos, platos y monumentos de Andalucía.
  • Funcionalidades de búsqueda y filtrado de la información.
  • Posibilidad de crear una cuenta de usuario y autenticarse en la aplicación.
  • Acceso a la información de usuario y posibilidad de actualizarla.
  • Funcionalidades de administración de usuarios (solo para administradores).
  • Funcionalidades de administración de logs (solo para administradores).

Gestión de usuarios

En este apartado se detalla cómo se gestionan los usuarios en la aplicación de Andalucía Descubre y a quién está destinado cada tipo de usuario. La aplicación de Andalucía Descubre cuenta con dos tipos de usuarios: usuarios normales, gestores de pueblos y administradores de la aplicación.

Usuarios

Son aquellos que tan solo visitan la aplicación para obtener información sobre los lugares más emblemáticos de Andalucía, los usuarios pueden acceder a la información de las localidades, eventos, platos y monumentos de Andalucía, así como buscar y filtrar la información según sus preferencias. Los usuarios no crearán cuentas, por lo que no tendrán acceso a la información de usuario ni a las funcionalidades de administración de usuarios y logs. Los usuarios pueden acceder a la información de la aplicación sin necesidad de autenticarse, pero no podrán realizar acciones que requieran autenticación.

Gestores de pueblo

Son aquellos que se encargan de gestionar la información de las localidades de Andalucía, los gestores de pueblo pueden crear, actualizar y eliminar la información de las localidades, eventos, platos y monumentos de Andalucía, así como buscar y filtrar la información según sus preferencias. Los gestores de pueblo pueden acceder a la información de la aplicación y realizar acciones que requieran autenticación, pero no podrán acceder a las funcionalidades de administración de usuarios y logs. Este tipo de cuenta irá destinada a los ayuntamientos y entidades locales que deseen promocionar los lugares más emblemáticos de sus localidades.

Administradores

Son aquellos que se encargan de gestionar la información de la aplicación de Andalucía Descubre, los administradores pueden crear, actualizar y eliminar la información de los usuarios, localidades, eventos, platos y monumentos de Andalucía, así como buscar y filtrar la información según sus preferencias. Los administradores pueden acceder a la información de la aplicación y realizar acciones que requieran autenticación, así como acceder a las funcionalidades de administración de usuarios y logs. Este tipo de cuenta irá destinada al equipo de desarrollo de la aplicación o al encargado de una entidad pública o privada que desee gestionar la información de la aplicación.

FAQs

En esta sección se detallan las preguntas frecuentes sobre la aplicación de Andalucía Descubre y se proporcionan respuestas a las dudas más comunes de los usuarios.

Preguntas de usuarios

¿Qué es Andalucía Descubre?

Andalucía Descubre es una aplicación web que permite a los usuarios descubrir los lugares más emblemáticos de Andalucía, como localidades, eventos, platos y monumentos, así como obtener información detallada sobre cada lugar.

¿Cómo puedo acceder a la información de Andalucía Descubre?

Puedes acceder a la información de Andalucía Descubre a través de la aplicación web, donde encontrarás la información de las localidades, eventos, platos y monumentos de Andalucía, así como la posibilidad de buscar y filtrar la información según tus preferencias.

¿Puedo crear una cuenta en Andalucía Descubre?

No, los usuarios no pueden crear cuentas en Andalucía Descubre, ya que la aplicación está destinada a proporcionar información sobre los lugares más emblemáticos de Andalucía de forma que solo los gestores de pueblo y los administradores pueden crear cuentas.

¿Cómo puedo contactar con el equipo de Andalucía Descubre?

Puedes contactar con el equipo de Andalucía Descubre a través de nuestro correo electrónico

Preguntas técnicas

¿Cómo puedo acceder a la API de Andalucía Descubre?

Puedes acceder a la API de Andalucía Descubre a través de los endpoints disponibles en la documentación de la API, donde encontrarás la información necesaria para realizar peticiones a la API y obtener la información de los lugares más emblemáticos de Andalucía. La url de la API es: https://andaluciadescubre.usuariozombie.com/

¿Qué tipo de autenticación utiliza la API de Andalucía Descubre?

La API de Andalucía Descubre utiliza tokens JWT (JSON Web Tokens) para proteger los endpoints de la API y garantizar que solo los usuarios autorizados puedan acceder a la información. Los tokens JWT son cadenas de texto codificadas que contienen información sobre el usuario autenticado, así como una firma digital que garantiza la integridad del token.

¿Necesito permiso para desarrollar una aplicación con la API de Andalucía Descubre?

No, no necesitas permiso para desarrollar una aplicación con la API de Andalucía Descubre, ya que la API es de uso público y está disponible para todos los desarrolladores que deseen acceder a la información de los lugares más emblemáticos de Andalucía.

¿Cómo puedo contribuir al desarrollo de la API de Andalucía Descubre?

Puedes contribuir al desarrollo de la API de Andalucía Descubre a través de nuestro repositorio en GitHub, donde encontrarás el código fuente de la API y podrás colaborar en su desarrollo.

Preguntas de gestores de pueblo

¿Cómo puedo gestionar la información de mi localidad en Andalucía Descubre?

Puedes gestionar la información de tu localidad en Andalucía Descubre a través de la aplicación web, debes registrarte como gestor de pueblo y ser verificado por un administrador para poder acceder a las funcionalidades de gestión de la información de tu localidad.

¿Qué tipo de información puedo gestionar como gestor de pueblo?

Como gestor de pueblo puedes gestionar la información de tu localidad, incluyendo la información de los eventos, platos y monumentos de tu localidad, así como buscar y filtrar la información según tus preferencias.

¿Cómo puedo promocionar mi localidad en Andalucía Descubre?

Puedes promocionar tu localidad en Andalucía Descubre a través de la aplicación web, gestionando la información de tu localidad y creando eventos, platos y monumentos que destaquen los lugares más emblemáticos de tu localidad.
